ใบงานปฏิบัติการ: การบำรุงรักษาเชิงพยากรณ์ (Predictive Maintenance) ด้วย IoT และ ML

วิชา: Internet of Things (IoT) สำหรับวิศวกรรมคอมพิวเตอร์

หัวข้อ: การใช้ปัญญาประดิษฐ์เพื่อทำนายอัตราการเสื่อมสภาพและอายุการใช้งานที่เหลืออยู่ (Remaining Useful Life: RUL) ของอุปกรณ์ระบายความร้อน

เป้าหมาย: เขียนระบบ Python AI Worker เพื่อดักฟังข้อมูลสหสัมพันธ์ผ่าน Apache Kafka ระหว่างพารามิเตอร์สภาพแวดล้อม (Temperature, Humidity, VPD) และข้อมูลทางไฟฟ้า/กายภาพของพัดลม (Current, Vibration) เพื่อนำมาทำนายโอกาสพัดลมชำรุดล่วงหน้า และบันทึกผลการพยากรณ์ลง InfluxDB สำหรับพล็อตบอร์ดเฝ้าระวังบน Grafana

1. ทฤษฎีและแนวคิดเชิงระบบ (Engineering & ML Concept)

ในระบบบำรุงรักษาเชิงพยากรณ์ (Predictive Maintenance) ของพัดลมระบายความร้อนตู้เซิร์ฟเวอร์ (Rack Cooling Fan) การรอให้อุณหภูมิสูงเกินไปแล้วค่อยแจ้งเตือน (Reactive) มักจะสายเกินไป เนื่องจากฮาร์ดแวร์จัดเก็บข้อมูลภายในอาจได้รับความเสียหายสะสมไปแล้ว

เราจะสร้างเซนเซอร์เสมือนเพื่อวิเคราะห์ "ความเสื่อมสภาพทางฟิสิกส์ (Degradation Physics)" ของตลับลูกปืนพัดลม (Bearing Wear) โดยใช้ข้อมูลสหสัมพันธ์ (Correlation) ดังนี้:

1.1 ปัจจัยสหสัมพันธ์ (Microclimate Correlated Metrics)

อุณหภูมิและความชื้นสะสม (Ambient stress): อุณหภูมิภายในตู้ที่สูงและความชื้นที่ต่อเนื่อง เร่งให้สารหล่อลื่นในตลับลูกปืนพัดลม (Bearing Grease) แห้งและเสื่อมสภาพเร็วขึ้น

กระแสไฟฟ้าพัดลม (Fan Current Draw - $I$): เมื่อลูกปืนเริ่มฝืดเนื่องจากความร้อนสะสม พัดลมจะต้องเค้นพลังงานมากขึ้นเพื่อรักษาความเร็วรอบ ส่งผลให้กินกระแสไฟฟ้าสูงขึ้นผิดปกติ มีลักษณะเกิดสัญญาณ Spike เล็กๆ ถี่ขึ้น

ความสั่นสะเทือน (Vibration - $a_z$): แกนมอเตอร์ที่ฝืดหรือแกว่งจะส่งแรงสั่นสะเทือนสะท้อนออกมาผ่านชิป IMU (MPU6050) บนบอร์ดพัฒนา

1.2 อัลกอริทึมปัญญาประดิษฐ์ (ML Classifiers / Regressors)

เราจะประยุกต์ใช้โมเดล Random Forest Classifier / Regressor (หรือ XGBoost ในระดับระบบจริง) เพื่อทำหน้าที่เรียนรู้ความสัมพันธ์ของข้อมูลคุณลักษณะ (Multi-feature Data) และส่งค่าทำนายออกมารูปแบบ "ความน่าจะเป็นที่พัดลมจะพัง (Failure Probability 0% - 100%)" และ "ระยะเวลาที่เหลือรอด (RUL: Remaining Useful Life)" เป็นชั่วโมงก่อนเกิดความเสียหายจริง

2. โค้ดสคริปต์สตรีมมิ่งบำรุงรักษาเชิงพยากรณ์ (Predictive Maintenance Worker)

สคริปต์ Python นี้จะทำหน้าที่ดักฟังข้อมูลสตรีมที่มีคีย์ข้อมูลผสมระหว่างสภาพแวดล้อมและเซนเซอร์กายภาพ นำมาวิเคราะห์ผ่านแบบจำลองปัญญาประดิษฐ์เพื่อทำนายผล และยิงข้อมูลวิเคราะห์เขียนกลับลง InfluxDB

## 3. การสร้างหน้าแดชบอร์ดสรุปตารางซ่อมบำรุงล่วงหน้าบน Grafana

เมื่อบันทึกผลการทำนายลง InfluxDB เรียบร้อยแล้ว อาจารย์สามารถสั่งให้นักศึกษาเขียนคิวรีดึงข้อมูลไปพล็อตหน้า แดชบอร์ด สำหรับช่างซ่อมบำรุงในโรงงาน (Facilities Dashboard) ได้ดังนี้ครับ:

### 3.1 ดึงประวัติระดับสุขภาพและการเสื่อมสภาพ (Health Trend - Time Series)
สร้างหน้าจอแสดงแนวโน้มสุขภาพพัดลมที่ลดลง เพื่อจับตาดูทิศทางการเริ่มพังชำรุด:
```javascript
from(bucket: "iot_data")
  |> range(start: v.timeRangeStart, stop: v.timeRangeStop)
  |> filter(fn: (r) => r["_measurement"] == "fan_predictive_maintenance")
  |> filter(fn: (r) => r["id"] == "ID_68123456789")
  |> filter(fn: (r) => r["_field"] == "fan_health_score" or r["_field"] == "fan_rul_hours")
  |> aggregateWindow(every: v.windowPeriod, fn: mean, createEmpty: false)
```

### 3.2 ทำป้ายสวิทช์สถานะใบงานแจ้งซ่อมบำรุงล่วงหน้า (Maintenance Dispatch Card)
สร้าง Panel ประเภท **Stat** ขวามือเพื่อแสดงพิกัดและลำดับความสำคัญของอุปกรณ์ที่ช่างต้องเดินไปซ่อมก่อนตามคิวงาน:
* **Flux Query:**
```javascript
from(bucket: "iot_data")
  |> range(start: -5m)
  |> filter(fn: (r) => r["_measurement"] == "fan_predictive_maintenance")
  |> filter(fn: (r) => r["id"] == "ID_68123456789")
  |> filter(fn: (r) => r["_field"] == "maintenance_priority")
  |> last()
```
* **ตั้งค่า Value Mappings ของพาเนลทางขวามือบน Grafana:**
  * `0` $\rightarrow$ แสดง **"NORMAL"** (หรือสีเขียว 🟢)
  * `1` $\rightarrow$ แสดง **"SCHEDULE REPAIR"** (สีเหลืองเฝ้าระวัง 🟡 - ใบงานจองคิวเปลี่ยนน้ำมันแบริ่งเมื่อครบรอบการรันระบบ)
  * `2` $\rightarrow$ แสดง **"🚨 EMERGENCY DISPATCH! (ชำรุดเฉียบพลัน)"** (สีแดงกะพริบวิกฤต 🔴 - ใบสั่งงานแจ้งช่างวิ่งเปลี่ยนพัดลมทันทีเพื่อป้องกันตู้เซิร์ฟเวอร์หลักช็อตเสียหาย)
